🔑 Aliases 관리

SSH 원격 서버와 API 키를 alias로 등록합니다. 실제 값은 암호화 저장되며 클라이언트에 노출되지 않습니다.

SSH Remotes
Search API Keys
SSH 원격 서버
🖥️
환경을 먼저 선택하세요.
보안 원칙: 클라이언트(LangGraph 등)는 alias 이름만 사용합니다. 실제 host/username/password/key는 서버 내부 암호화 저장소에만 존재합니다. 예: ssh_run(alias="remote1", cmd="ls")
Search API Keys
🔍
환경을 먼저 선택하세요.
비용 가드: API 키는 암호화 저장됩니다. 클라이언트는 alias만 사용합니다. 쿼터 초과 시 서버가 자동으로 요청을 차단합니다. 예: web_search(alias="tavily_default", query="...")

🛡️ 정책 설정

모든 정책은 설정이 아니라 강제(enforcement)됩니다. 클라이언트가 우회할 수 없습니다.

⚙️ Exec 정책 항상 강제
추천 이유: 무한 대기/블로킹 방지. 실패 징후는 보통 빠르게 드러남. 값을 높이면 장시간 블로킹 위험이 증가합니다.
추천 이유: 로그 폭주로 인한 비용·응답 지연·저장 공간 문제 방지.
추천 이유: 로그/상태 혼선 및 자원 폭주 방지. 순차 실행이 기본.
추천 이유: cwd를 고정하여 임의 경로 접근을 방지합니다.
추천 이유: 위험한 시스템 명령을 기본 차단합니다.
📁 Filesystem 정책 항상 강제
추천 이유: 이 경로 외부 접근은 심볼릭 링크 우회 포함 모두 차단됩니다.
기본 OFF — 읽기 전용. 쓰기는 명시적으로 활성화해야 합니다.
추천 이유: 기본적으로 읽기 전용. 쓰기를 활성화하면 파일 변경/삭제 위험이 생깁니다.
🔍 Search 정책 (Tavily) 항상 강제
추천 이유: 근거 다양성 확보 + 비용 통제의 균형. 값을 높이면 API 비용이 증가합니다.
추천 이유: pay-as-you-go에서 비용 폭탄 방지.
추천 이유: 예산 초과 시 서버가 자동으로 요청을 차단합니다.
추천 이유: basic이 비용 대비 효율이 좋고 대부분의 조사에 충분합니다.
📋 Logs 정책
추천 이유: 관측은 충분히 하되 전체 덤프로 폭주하지 않도록.
추천 이유: 최근 이슈 진단에 충분한 시간 범위.
